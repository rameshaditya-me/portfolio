LLM Evaluation Pipeline for Reasoning Tasks

An automated pipeline for evaluating large language models on chain-of-thought and tool-use benchmarks with reproducibility and reporting.

Problem

Evaluating LLMs on reasoning and tool-use benchmarks requires running many prompts, parsing outputs, and aggregating metrics. Doing this in a reproducible way across model versions and datasets is error-prone when scripts are one-off.

Why the problem is hard

Output formats vary by model and task; grading often needs a judge model or regex; and results should be versioned with the model and dataset for comparison.

System design

We use a DAG-based pipeline (config-driven): load dataset → generate responses (with caching) → run graders → aggregate and write reports. All steps are deterministic and keyed by (model_id, dataset_version, prompt_template).

Implementation details

Python + YAML configs; parallel execution with a process pool; results stored in SQLite for quick comparison across runs.

Experiments

We run on three reasoning benchmarks and two tool-use benchmarks; report accuracy and cost per run.

Results

Pipeline reduces evaluation time from ad-hoc scripts by ~5x and ensures every reported number is reproducible from a config and commit.

Takeaways

Config-driven pipelines and caching are essential for reliable LLM evaluation; invest in a small schema for (model, dataset, metric) early.
