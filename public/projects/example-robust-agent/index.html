Robust Multi-Task Agent in Simulation

A single policy that generalizes across multiple simulated environments with different dynamics and reward scales.

Problem

Training one agent that works across several related but different environments (e.g. different robot morphologies or reward scales) often fails due to negative transfer or one task dominating the gradient.

Why the problem is hard

Task balance, representation sharing, and distribution shift at test time make multi-task RL difficult without careful design.

System design

We use a shared encoder and task-specific heads, with a gradient-based task weighting (Uncertainty Weighting) and normalized rewards per task.

Implementation details

PyTorch; PPO base; 4 simulation tasks; training time ~12h on 8 CPUs.

Experiments

Ablations: single-task baselines, shared vs. separate encoders, with and without task weighting.

Results

Multi-task agent matches or exceeds the average of single-task agents on 3/4 tasks with a single policy.

Takeaways

Reward normalization and task weighting are as important as architecture for multi-task RL; start with a small set of tasks before scaling.
