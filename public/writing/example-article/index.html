Goal

We walk through a minimal implementation of Conservative Q-Learning (CQL) for offline RL. The key idea is to penalize the Q-function for assigning high values to actions that are unlikely under the behavior policy, reducing overestimation.

Main equation

The CQL loss adds a regularizer to the standard TD loss:

$$ \mathcal{L}{\mathrm{CQL}} = \alpha \cdot \mathbb{E}{s \sim \mathcal{D}}\left[ \log \sum_a \exp(Q(s,a)) - \mathbb{E}{a \sim \pi\beta(a|s)}[Q(s,a)] \right] + \mathcal{L}_{\mathrm{TD}} $$

In practice we sample states from the buffer, actions from the policy, and use a Monte Carlo estimate.

Code sketch

CQL penalty term (PyTorch)


def cql_penalty(q_values, log_probs):
    # q_values: (B, A), log_probs: (B, A)
    log_sum_exp_q = torch.logsumexp(q_values, dim=-1)  # (B,)
    mean_q_under_pi_b = (q_values.exp() * log_probs.exp()).sum(dim=-1)  # simplify
    return (log_sum_exp_q - mean_q_under_pi_b).mean()

Note

This is a simplified version; the full CQL paper uses importance sampling and multiple samples. See the paper for details.

Summary

  • CQL penalizes Q for being large on out-of-distribution actions.
  • Implementation is a small addition on top of a standard DQN/SAC training loop.
  • Tune the coefficient (\alpha) on a validation set to balance return and stability.
